Welcome to the uim-controller sublibrary.

Controllers represent the "C" in the MVC pattern. After the correct controller has been found using routing, the controller's action is called. The controller should handle processing the request data and ensure that the correct models are called and the correct answers or views are rendered. Controllers are therefore the controlling layer between the model and the view.

Typically, a controller is used to manage logic around a single model. For example, an ordering website might have a corresponding controller such as OrderController that manages orders.
